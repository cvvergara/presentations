The interest is on the geometry of the route ...

Make a view

  1. Join the results with edge_table
  2. Extract the geometry
  3. "seq" will give us the unique id (QGIS requisite)

CREATE VIEW route_from2To3_1 AS
WITH
dijkstra AS (SELECT * FROM pgr_dijkstra('
    SELECT id,
        source, target,
        cost, reverse_cost
        FROM edge_table',
    2, 3, true))
SELECT seq, the_geom AS route_geom FROM dijkstra
   JOIN edge_table ON(edge = id);

See the result in Qgis

  1. Add new layer route_from2To3_1
  2. Hide Edges layer

After adding arrows to the route_from2To3_1 layer

Some arrows indicate the "wrong" direction

  • From 4 to 9 should be from 9 to 4
  • From 3 to 4 should be from 4 to 3
There is the need to temporarly flip the geometries!!
The orientation of the geometries depend on the node from where the route's edge starts
Use the node column of the results to adjust the geometry

Pseudocode 


if (The node is the source on the edge_table)
    The orientation of the geometry is ok
Else
    Reverse the orientation of the geometry

Code 


CASE
WHEN dijkstra.node = edge_table.source
  THEN edge_table.the_geom
  ELSE ST_Reverse(edge_table.the_geom)
END AS route_geom

Make a new view 


CREATE VIEW route_from2To3_2 AS
WITH
dijkstra AS (SELECT * FROM pgr_dijkstra('
    SELECT id, source, target, cost, reverse_cost
        FROM edge_table',
    2, 3, true))
SELECT seq,
    CASE
    WHEN dijkstra.node = edge_table.source
      THEN edge_table.the_geom
      ELSE ST_Reverse(edge_table.the_geom)
    END AS route_geom
FROM dijkstra
   JOIN edge_table ON(edge = id);

See the result in Qgis

  1. Add new layer route_from2To3_2
  2. Add arrows to route_from2To3_2 layer
  3. Hide layer route_from2To3_1

When additional information is needed

For example, the id of the geometry...

Make a new view 


CREATE VIEW route_from2To3_3 AS
WITH
dijkstra AS (SELECT * FROM pgr_dijkstra('
    SELECT id, source, target, cost, reverse_cost
        FROM edge_table',
    2, 3, true))
SELECT seq, id AS name, -- additional information
    CASE
    WHEN dijkstra.node = edge_table.source
      THEN edge_table.the_geom
      ELSE ST_Reverse(edge_table.the_geom)
    END AS route_geom
FROM dijkstra
   JOIN edge_table ON(edge = id);

See the result in Qgis

    1. Add new layer route_from2To3_3
    2. Add arrows to route_from2To3_3 layer
    3. Label each segment with name
    4. Hide layer route_from2To3_2
